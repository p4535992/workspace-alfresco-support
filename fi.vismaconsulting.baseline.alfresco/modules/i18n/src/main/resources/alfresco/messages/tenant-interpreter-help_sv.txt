##
##  Meta kommandon
##

ok> help

    Listar denna hjälp

ok> r

    Repetera senaste kommando.

##
##  Quit / Exit
##

    Om du kör en webbläsare, stäng detta fönster eller tab (och logga ut från Alfresco Explorer eller stäng webbläsare).
    
    Om du kör kommand-radsverktyg, skriv 'quit' eller 'exit'.

##
##  Tenant Kommandon - för att administrera tenants
##


ok> show tenants

    Listar information om alla tenants.
    
ok> show tenant <tenant domain>

    Visa information om tenant - status (dvs. om den är aktiverad eller inaktiverad) och root content store directory.

    Exempel:   show tenant yyy.zzz.com

ok> create <tenant domain> <tenant admin password> [<root contentstore dir>]

    Skapa tom tenant. Från skapande kommer tenant vara aktiverad. Det kommer ha en admin
    användare kallad admin@<tenant domain> med angivet admin lösenord. Alla användare
    som admin skapar kommer logga in med <username>@<tenant domain>. 
    Roten för contentstore-sökvägen kan frivilligt anges, annars 
    så kommer det att som standard använda defautl root contentstore (såsom angivet 
    i dir.contentstore property). Standard arbetsflöden kommer också att bli bootstrapped.
    
    Exempel:  create zzz.com l3tm31n /usr/tenantstores/zzz
               create yyy.zzz.com g00dby3 /usr/tenantstores/yyy.zzz
               create myorg h3ll0

ok> changeAdminPassword <tenant domain> <tenant admin password>

    Användbart om tenant's admin (admin@<tenant domain>) har glömt sitt lösenord.
    
    Exempel:   changeAdminPassword yyy.zzz.com n3wpassw0rd

ok> enable <tenant domain>

    Aktivera tenant så att det är tillgängligt för login
    
    Exempel:   enable yyy.zzz.com

ok> disable <tenant domain>

    Inaktivera tenant så det är inaktivt Existerande login kommer misslyckas nästa gång de används.
    
    Exempel:   enable yyy.zzz.com
    
ok> delete <tenant domain>

	BETA - Radera tenant. 
	
	Notera: Detta kräver för närvarande en omstart av servern för rensing av indextrådar. Dessutom
	tenant index folders måste raderas manuellt.
	
	Exempel:   delete yyy.zzz.com
    
ok> export <tenant domain> <destination directory>

    Exportera tenant till given plats. Export filnamn kommer att ha suffix '<tenant domain>_'.
    
    Exempel:   export yyy.zzz.com /usr/exportdir
    
ok> import <tenant domain> <source directory> [<root contentstore dir>]

	BETA - skapa en tenant genom import av tenant filer från angiven källmapp. Importfilnamnet måste ha suffix '<tenant domain>_'.
	
	Notera: Om import sker till tidigare raderad tenant måste server stoppas efter radering
	(och tenant indexes manuellt raderade) före omstart och utförande av imort.
	
	Exempel:   import yyy.zzz.com /usr/exportdir /usr/tenantstores/yyy.zzz
